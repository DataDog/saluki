9/// An ergonomic wrapper around fairly utilizing I/O buffers from an object pool.
10///
11/// As I/O buffer pools are fixed-size for the DogStatsD source, this presents an issue when looking to handle more
12/// connections than there are I/O buffers. If each new connection was simply to acquire a buffer from the pool and then
13/// reuse it until the connection closed, all new connections made after the buffer pool was exhausted would be blocked
14/// on acquiring an I/O buffer, potentially indefinitely. This would occur even if the existing connections were idle.
15///
16/// `IoBufferManager` provides a simple, ergonomic wrapper over a basic pattern of treating the current buffer as
17/// optional, which allows the wrapper to release the current buffer back to the pool, and acquire a new one, all before
18/// returning a reference to the buffer. This provides fairness by ensuring that tasks which are waiting for a buffer
19/// can eventually acquire one once existing tasks are able to reach a consistent point that they can release their
20/// buffer.
21///
22/// ## Release behavior
23///
24/// This wrapper provides two basic behaviors:
25///
26/// - acquire a new buffer when the current buffer does not exist
27/// - retain the current buffer (and collapse it) if there is remaining data, _or_ release it if there is no remaining data
28///
29/// When the current buffer still has remaining data, we must preserve the buffer and its data as we could otherwise be
30/// throwing away a partial frame that will be fulfilled by the next socket read. We additionally handle collapsing the
31/// current buffer when there is remaining data, as this ensures that all available capacity is contiguous and directly
32/// follows whatever remaining data exists.
33///
34/// When the current buffer has no remaining data, we can safely release it back to the pool prior to immediately
35/// reacquiring a new buffer.
36///
37/// ## Connectionless streams
38///
39/// For connectionless streams, the current buffer is never released and is always reused. This is because
40/// connectionless streams are often higher volume in terms of the number of socket reads that occur, and are a fixed
41/// number that does not change with the amount of traffic that is being sent. This allows us to reduce the overhead
42/// related to acquiring and releasing buffers, ensuring that these inherently lossy streams are not introducing
43/// additional overhead that might cause unnecessary packet loss.
44pub struct IoBufferManager<'a, O>
45where
46    O: ObjectPool<Item = BytesBuffer>,
47{
48    pool: &'a O,
49    should_retain: bool,
50    current: Option<BytesBuffer>,
51}
52
53impl<'a, O> IoBufferManager<'a, O>
54where
55    O: ObjectPool<Item = BytesBuffer>,
56{
57    /// Creates a new `IoBufferManager` for the given stream and object pool.
58    pub fn new(pool: &'a O, stream: &Stream) -> Self {
59        Self {
60            pool,
61            should_retain: stream.is_connectionless(),
62            current: None,
63        }
64    }
65
66    /// Returns a mutable reference to the current buffer.
67    ///
68    /// This method may or may not release the current buffer depending on if remaining data exists or not. When no
69    /// buffer is available (including after intentionally releasing it), a new buffer will be acquired before
70    /// returning.
71    pub async fn get_buffer_mut(&mut self) -> &mut BytesBuffer {
72        // Consume the current buffer, if it exists.
73        let current = self.current.take().and_then(|mut buffer| {
74            if buffer.has_remaining() || self.should_retain {
75                // Collapse the remaining data in the buffer and continue using it.
76                buffer.collapse();
77                Some(buffer)
78            } else {
79                // Release the buffer back to the pool.
80                None
81            }
82        });
83
84        // Determine our replacement buffer: either we're sticking with the current buffer, or reacquiring a new one.
85        let new_current = match current {
86            Some(buffer) => buffer,
87            None => {
88                let new_buffer = self.pool.acquire().await;
89                trace!(
90                    remaining = new_buffer.remaining(),
91                    capacity = new_buffer.capacity(),
92                    "Acquired new buffer from pool."
93                );
94
95                new_buffer
96            }
97        };
98
99        // Finally, store the new buffer and reference a mutable reference to it.
100        self.current = Some(new_current);
101        self.current.as_mut().unwrap()
102    }
103}
